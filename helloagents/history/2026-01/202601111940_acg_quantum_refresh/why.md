# 变更提案: ACG Quantum Refresh

## 需求背景
当前站点已具备多源聚合、搜索与收藏能力，但在信息密度、可视化引导与高频用户效率层面仍有提升空间。用户需要更快识别“正在爆发的信号”、更少步骤完成筛选，并在高密度内容下保持视觉舒适与性能稳定。同时，现有布局与视觉语言偏保守，难以承载“未来感/行业顶尖审美”的目标定位。

## 产品分析

### 目标用户与场景
- **用户群体:** ACG 资讯重度用户、轻度追踪用户、移动端阅读用户
- **使用场景:** 早晚快速扫热点、通勤碎片阅读、深度阅读与收藏管理
- **核心痛点:** 热点识别成本高、同质内容噪音、来源可靠性不直观、视觉疲劳

### 价值主张与成功指标
- **价值主张:** 以信号化、分层化与低干扰界面提升发现效率与阅读舒适度
- **成功指标:**
  - 首页热点识别时间显著下降
  - 常用筛选步骤减少
  - Lighthouse 指标稳定提升（性能/可访问性/最佳实践）
  - 视觉一致性与信息密度兼容提升

### 人文关怀
- 明确提示内容版权归原站
- 保持“本地优先”数据与设置，不上传隐私
- 提供 reduced-motion 与对比度友好样式

## 变更内容
1. 构建“信号层”指标：热度/新鲜度/阅读深度/来源健康度
2. 扩展筛选与排序：时间透镜、稳定来源过滤、去重视图、按热度排序
3. UI/交互重塑：未来感视觉系统、信息分层、沉浸式布局
4. 性能与可用性优化：渲染策略、动效节制、视觉稳定性
5. 双语文档与知识库同步升级

## 影响范围
- **模块:** pages / components / styles / client / lib / docs
- **文件:** src/pages、src/components、src/styles、src/client、README.md、helloagents/wiki/*、helloagents/CHANGELOG.md
- **API:** 无新增外部 API
- **数据:** 新增本地派生指标，不改变上游同步数据协议

## 核心场景

### 需求: Pulse Ranking
**模块:** 首页/洞察
以“热度评分”突出高价值内容，降低热点识别成本。

#### 场景: Hot Scan
用户进入首页时希望迅速看到最热内容。
- 显示热点排行与脉冲评分
- 支持一键切换为热度排序

### 需求: Time Lens Filters
**模块:** 客户端筛选
通过时间透镜快速定位“最新/近期”内容。

#### 场景: Time Focus
用户只关注近 2h/6h/24h 的内容。
- 一键切换时间窗口
- 过滤结果立即反馈

### 需求: Smart Dedup
**模块:** 列表展示
减少相似标题的重复噪音。

#### 场景: Noise Control
多来源转载导致重复内容堆叠。
- 同类内容合并/折叠
- 保留可展开入口

### 需求: Read Depth
**模块:** 卡片信息层
为用户提供阅读成本预估。

#### 场景: Read Plan
用户希望区分“快速扫/深度读”。
- 展示阅读时长
- 可按阅读深度筛选

### 需求: Source Trust
**模块:** 来源标识
直观呈现来源健康度与稳定性。

#### 场景: Trust Check
用户在点击前希望快速判断来源稳定性。
- 展示健康度等级
- 支持“只看稳定来源”

### 需求: Immersive UI
**模块:** 全站视觉
提升信息密度与审美水准。

#### 场景: Home Discovery
用户希望在高密度信息中保持舒适与清晰。
- 未来感布局与分层视觉
- 动效克制且可被关闭

## 风险评估
- **风险:** UI 与交互变更过大导致学习成本上升
  - **缓解:** 维持核心信息架构与关键交互位置，提供清晰引导
- **风险:** 新筛选/排序逻辑影响性能
  - **缓解:** 预计算派生指标、限制计算频次、使用虚拟化策略
- **风险:** 本地存储结构升级导致偏好丢失
  - **缓解:** 提供向后兼容迁移与默认值填充
