# 变更提案: Telemetry 自助排障（导出/清空 + 统计）

## 背景

项目采用“静态站点 + GitHub Actions 定时同步 + GitHub Pages 部署”的架构，默认不引入后端常驻服务。随着端侧功能增强（收藏/已读/过滤/全文预览/可观测性等），偶发问题更需要**可回溯线索**来辅助排障，同时必须保持**隐私可控**与**零后端依赖**。

当前已有本地优先 telemetry（默认仅写入 localStorage，用户显式开启后可上报），但缺少一个“自助排障”的行业标准入口：
- 无法直观看到本地记录量/占用空间
- 无法一键导出给维护者分析
- 无法一键清空以重置状态或保护隐私

## 目标与成功标准

- 在不破坏核心架构与既有逻辑根基的前提下，为偏好面板的 Telemetry 模块补齐**导出/清空 + 统计**。
- 默认仍保持“本地记录、不上传”，所有操作由用户显式触发。
- `astro check` / `npm test` / `npm run build` / `npm run budget` 通过。

## 非目标

- 不新增第三方监控 SDK。
- 不引入新的后端服务或自动上报逻辑。
