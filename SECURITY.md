# 安全政策 / セキュリティポリシー

如果你发现了安全漏洞或涉及敏感信息的问题，请不要在公开 Issue 中披露。
脆弱性や機微情報を見つけた場合、公開 Issue での報告は避けてください。

---

## 报告方式 / 報告方法

推荐方式（优先）/ 推奨（優先）：

- 使用 GitHub 的 **Security Advisories**（私密报告流程）。
  - GitHub の **Security Advisories**（非公開）をご利用ください。

备选方式 / 代替：

- 若无法使用 Advisory，请创建 Issue，但**不要包含**漏洞利用细节、密钥或个人信息，并 @ 维护者说明需要私下沟通。
  - Advisory が使えない場合は Issue を作成できますが、**悪用手順や秘密情報は書かない**でください。必要に応じてメンテナに @ してください。

---

## 支持的版本 / サポート対象

本项目为静态站点与定时同步项目，通常仅维护 `main` 分支与最新部署版本。
本プロジェクトは静的サイトのため、原則として `main` と最新デプロイを対象にします。

---

## 披露原则 / 公開の原則

- 我们会在修复完成后再公开细节。
  - 修正後に詳細を公開します。
- 若问题影响范围较大，可能会先给出缓解建议再发布补丁。
  - 影響が大きい場合、先に回避策を提示することがあります。
